# WebChat：可以帮你理解和分析当前网页的内容的AI助手

**【完全开源】** 基于Chrome扩展的最佳实践开发，AI助手能够基于当前网页内容进行智能问答。支持自定义API和本地大模型。

## 项目说明

本项目基于 [Airmomo/WebChat](https://github.com/Airmomo/WebChat) 二次开发。

### v1（原项目基础上新增）
- 会话日志自动落地到本地 Markdown 文件
- 本地日志桥接服务与失败重试机制
- 会话切分、恢复与并发生成控制
- 默认配置与文档的开源清理

### v2（本版本，`tag: v2`）
- 新增"选中 + 临时"会话模式：只把页面选中文本作为上下文，不注入整页
- 全局快捷键 `Cmd/Ctrl+Shift+K` 一键打开/关闭侧边面板
- 面板内快捷键：`Enter` 发送 / `Shift+Enter` 换行 / `Cmd+Enter` 也可发送 / `Esc` 生成时停止否则关闭面板
- 输入框上方"选中内容 Chip"：页面一选中就显示，点击即以 Markdown 引用插入；或输入 `@sel` + 空格展开
- `/commands` 提示词模板菜单：输入 `/` 弹出 10 条预置模板（总结 / TL;DR / 翻译 / 出题 / 改写 等），方向键+Tab/Enter 导航
- 按域名记忆默认会话模式：某域名上改过一次，同域名后续新标签自动用同一模式
- 头部新增按钮：🎓 学习带教模式（苏格拉底式引导，算法/Python/费曼/通用四种风格）、📍 本页关键概念自动标注（悬停看释义）、🧹 一键清空当前会话、⤢ 宽度预设循环（360/420/560/760）、⇄ 左右停靠
- 学习加强：选中 AI 回答中任意一句 → 浮出 **🔍 追问这句** 按钮，一键生成追根究底式追问
- `/commands` 新增 `/quiz` 自测、`/feynman` 费曼一次性、`/deepdive` 针对上一句深挖
- 底层重构：抽取 `shared/chatModes.js`、`shared/mentorModes.js`，避免 content/popup/background 三处枚举漂移

### 🎓 学习带教模式（v2 新增）

点击面板头部的 🎓 按钮，选择一种带教风格，让 AI 从"直接回答"切换到"苏格拉底式引导"：

- **算法带教**：按"理解题目 → 暴力解 → 优化解 → 总结"六步引导，适合 LeetCode / 算法题网页
- **Python 带教**：对比 JS → Python，5 步法，适合从前端/已会语言切 Python
- **费曼带教**：**你**讲给 AI 听，AI 扮演"完全不懂的小白"追问你每个含糊处，连续讲清 3 轮后给点评
- **通用带教**：不限学科，按"测试理解 → 补充概念 → 小练习 → 总结"循环，适合读论文/文档/教程
- **不使用**：回到普通问答

### 📍 网页知识点自动标注（v2 新增）

点击面板头部的 📍 按钮，AI 会读当前页面正文，识别 8-15 个关键概念/术语，直接在页面上**黄色下划线高亮**，鼠标悬停即可看一句话口语化释义。再点一次 📍 清除。

- 自动跳过代码块 `<pre>` / `<code>` / `<script>` / 表单控件等
- 每个术语最多高亮 2 次，避免页面被淹没
- 和会话模式独立：标注不会消耗对话上下文

### 🔍 追根究底（v2 新增）

在任何一条 AI 回答里用鼠标选中一段文字 → 右上浮出 **🔍 追问这句** 按钮 → 点击会把这段自动以 Markdown 引用插到输入框，并追加"请讲透 (1) 精确含义 (2) 为什么 (3) 一个最小例子 (4) 常见误解"。

### 🎯 快速自测 / 🧠 一次性费曼 / 🔍 深挖

在输入框打 `/` 唤起命令面板，试试 `/quiz`、`/feynman`、`/deepdive`。

特点：

- 与现有 5 种会话模式**正交叠加**（如 `选中 + 临时` + 算法带教 = 在 LeetCode 题页贴着题面做苏格拉底训练）
- AI 严守"不在学生思考之前直接给答案"等教学纪律
- 学生用关键词控制节奏：`下一步` / `给提示` / `答案` / `总结`
- **提示词完全可编辑**：设置页 → "🎓 学习带教模式提示词" 展开任一模式，直接改、改完即时生效；清空输入框恢复默认

## 安装方法

1. 下载项目代码：`https://github.com/KitschYoung/webchat.git`
2. 打开Chrome扩展管理页面 (chrome://extensions/)
3. 开启"开发者模式"

## 基本使用说明

- 插件开启后刷新页面会显示一颗悬浮球，点击它打开对话框
- 输入问题后点击发送并等待AI回答
- 右键点击历史消息可复制内容

## 会话日志保存

扩展现在支持将每次会话实时同步到本地目录，默认目标是：

`~/webchat-session-logs`

由于 Chrome 扩展不能直接写任意本地绝对路径，需要先启动一个本地桥接服务：

```bash
cd /path/to/WebChat
python3 tools/session_log_bridge.py
```

默认服务地址：

`http://127.0.0.1:8765/log-session`

默认会写出结构化 Markdown 文件，内容包含：

- 页面标题、URL、域名、页面文本摘录
- 会话 ID、开始/更新时间、模型与上下文配置
- 完整的用户/助手消息时间线

启动桥接服务后，在扩展设置页中可直接测试日志服务连通性。

### 五种会话模式

popup 和悬浮窗内都提供同一组会话模式选择：

- `网页 + 入库`
- `网页 + 临时`
- `选中 + 临时`（v2 新增）
- `纯聊 + 入库`
- `纯聊 + 临时`

解释：

- `网页`：会把当前网页正文注入给模型
- `选中`：只把页面上用户已选中的文字注入给模型，不注入整页；未选中时发送会拦截并提示
- `纯聊`：不注入任何网页内容，只做普通聊天
- `入库`：该回合允许写入 Markdown 日志
- `临时`：该回合不会写入 Markdown，也不会进入待补写队列

模式切换不会清空当前聊天界面；切换后只影响后续新回合。  
同一可见会话中如果混合了"入库"和"临时"回合，日志文件只会保留允许入库的回合，中间被跳过的临时回合会以"已省略未入库回合"的提示标记。

**按域名记忆**：你在某个域名上切换过会话模式后，偏好会保存到 `chrome.storage.sync`，之后在同域名的新标签会自动使用这一模式（跨设备同步）。

### 快捷键与输入增强（v2）

| 快捷键 | 位置 | 作用 |
|---|---|---|
| `Cmd/Ctrl+Shift+K` | 任何网页 | 打开/关闭侧边面板（可在 `chrome://extensions/shortcuts` 改键位） |
| `Enter` | 面板输入框 | 发送 |
| `Shift+Enter` | 面板输入框 | 换行 |
| `Cmd/Ctrl+Enter` | 面板输入框 | 发送（在多行输入时更顺手） |
| `Esc` | 面板输入框 | 正在生成则停止生成；否则关闭面板 |
| `↑ / ↓` | `/commands` 菜单打开时 | 在模板列表中移动光标 |
| `Tab` 或 `Enter` | `/commands` 菜单打开时 | 应用选中模板 |
| `Esc` | `/commands` 菜单打开时 | 关闭菜单 |

**选中内容引用**：在页面上用鼠标选中一段文字（≥2 字），输入框上方会出现一个蓝色 chip 显示字符数，点击它就会把引用以 `> xxx` 形式插入到输入框；你也可以手动输入 `@sel ` 或 `@selection `（注意末尾空格）来展开成引用。

**/commands 提示词模板**：输入框里打一个 `/` 就会浮出模板菜单，内置：

| 命令 | 说明 |
|---|---|
| `/summarize` | 五条要点总结 |
| `/tldr` | 一句话概括（≤40 字） |
| `/translate-zh` | 翻译为中文 |
| `/translate-en` | 翻译为英文 |
| `/explain` | 通俗讲解 |
| `/outline` | 生成 Markdown 大纲 |
| `/keypoints` | 抽取 5-8 个核心要点 |
| `/qa` | 基于上文出 5 道练习题 |
| `/rewrite` | 改写得更清晰凝练 |
| `/counter` | 反驳/质疑角度 |

继续输入字符可过滤菜单（匹配命令名前缀或标题子串）。

### 头部按钮（v2）

- 🧹 **清空会话**：归档当前会话为 `.md`，面板清空，立即开始新会话
- ⤢ **宽度循环**：360 → 420 → 560 → 760 → 回到 360，记忆到 `panelWidth`
- ⇄ **左右停靠**：把面板从左边换到右边或反之
- × **关闭**：关闭面板（不结束会话）

### 会话切分规则

- 不同标签页：始终是不同会话，不会串到同一个日志文件
- 同一标签页切换到新 URL：会结束旧会话，并在下一次提问时生成新文件
- 同一标签页长时间未提问：超过设置页里的“空闲多久切分为新会话”后，再次提问会新建文件
- 同一标签页同时只能有一个正在生成的回复；如果另一个入口（popup / 悬浮球）同时提问，会被明确拦截
- 手动停止回复会真正中断后台请求，不会继续偷偷生成
- 如果日志桥接服务临时没启动，会话会先进入本地待重试队列，后续恢复服务后自动补写
- “工作区根目录” 仅用于补充日志元数据，可按自己的目录结构填写

## 支持的API

### 自定义API
- 支持OpenAI兼容的API接口
- 可配置自定义端点
- 出于开源发布考虑，默认不预填 API 密钥、请求 URL 和模型名称，需要首次使用时手动填写

### Ollama本地部署开源模型
1. 从[官网](https://ollama.ai)下载安装 Ollama
2. 设置允许跨域并启动：
- macOS：
```bash
launchctl setenv OLLAMA_ORIGINS "*"
```
- Windows：
1. 打开控制面板-系统属性-环境变量
2. 在用户环境变量中新建：
- 变量名：`OLLAMA_HOST`，变量值：`0.0.0.0`
- 变量名：`OLLAMA_ORIGINS`，变量值：`*`
- Linux：
```bash
OLLAMA_ORIGINS="*" ollama serve
```
3. 安装模型：
```bash
ollama pull qwen2.5
```
4. 启动或重启Ollama服务
```bash
ollama serve
```
5. 注意事项
可使用自定义 API 接口地址来请求ollama服务：
`http://localhost:11434/v1/chat/completions` 
如果你是在局域网内其他主机运行的ollama服务，那么请将localhost替换为你的主机IP地址。

## 注意事项

1. 使用自定义API需要配置API密钥
2. 使用本地模型需要先安装并启动ollama，并确保服务在后台运行
3. 确保网络连接正常
4. 网页内容仅用于当前对话，刷新页面后对话的内容以及历史记录会丢失
